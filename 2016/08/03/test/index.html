初步搭建好了！

自从在微博上看到笑来在写关于编程方面的教程，就想着赶紧看起来，趁着暑期时间多。当我发现笑来原来在博客上写《人人都是工程师》这本书，顿时想到那本《人人都能用英语》，不过我到现在也没怎么好好用过英语。我想是学英语学得太长时间了，对英语已经不抱着太大的热情，况且自身的基础会让态度变得傲慢，使效果大打折扣。毕竟在你还不算入门的情况下，踏踏实实地跟着初级教程做一遍，是比你自己探索来得更有效的。
既然有了热情和教程，剩下的就是全然按照教程进行。其实前面还有一个就是客观条件，比如要用到OS X系统和一台外接显示器。这可能会让你觉得没必要，但由于我常常看笑来的文章，他经常提到苹果系统能做些什么提高效率的事，比如[全文检索]（http://b.xinshengdaxue.com/A30.html），我也在几个月前在我的pc机上装了虚拟机与配置了第二块屏幕。记得刚配置完这些硬件，周围人都以为我要弄什么正经的研究，结果我其实没花多少时间做事，颇为愧疚。我的确认为应该有一定的客观设备才好进行模仿重复，但你没足够的动力是不行的。客观条件只能节省你心力上所关注的事，就好像你为啥要跑到健身房去跟着教练锻炼，因为你这样就不用自己在方式上想太多，只用好好练就行了。但是切不可认为只要满足了客观条件就能自动变成你想要的生活状态。

遇到的问题及解决方法

虽然教程一步步写得很详细，但是难免会出现问题，以至于我进行不下去。虽然可能出现的问题你基本都能google到，但我这里还是想稍稍总结一下经验，希望能补充补充。

安装nvm

在《基本开发环境设置》这里，我出现的问题是装不了nvm，出现的错误是nvm:command not found。这个错误在老师博客的评论那里也有提到，解决方法是要向~/.bash_profile添加两行代码。

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm

至于怎么添加，当时我试过open ~/.bash_profile和touch ~/.bash_profile都不行，最后用pico ~/.bash_profile才设定好nvm环境参数。

部署到GitHub

这个就比较简单了，记得看清楚有没有空格就可以了，少了空格的话就部署不了了。

atom .
deploy:
  type: git
  repo: https://github.com/yourname/yourname.github.io.git
